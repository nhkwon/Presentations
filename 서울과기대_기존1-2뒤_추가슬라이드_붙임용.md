# 서울과기대 기존 1, 2장 뒤 추가 슬라이드 붙임용 초안

전제:

- 기존 1장 표지 유지
- 기존 2장 Contents 유지
- 아래 슬라이드를 기존 2장 바로 뒤에 붙임
- 이후 기존 학력소개 및 BIM 기반 가상건설 슬라이드는 그대로 이어짐
- 발표시간 20분 이내를 위해 추가 슬라이드는 5~6장으로 제한

## 붙임 전략

기존 자료의 본문은 BIM 기반 가상건설과 건설관리 내용이 중심이다. 따라서 기존 2장 뒤에 스마트시티 내용을 먼저 넣어, 이후 BIM 내용이 `스마트시티 구현을 위한 기반기술`로 읽히게 만드는 것이 핵심이다.

추가 슬라이드의 역할은 다음과 같다.

```text
1. 스마트시티에서 왜 건설관리가 중요한지 제시
2. 건설관리가 기획-설계-시공-운영 생애주기를 연결한다는 점 설명
3. BIM, AI, Digital Twin, IoT 데이터가 어떻게 연결되는지 설명
4. 기존 BIM 강의 내용으로 자연스럽게 넘어갈 연결고리 제공
5. 본인의 연구와 향후 발전방향까지 연결
```

## 추가 슬라이드 구성

```text
추가 1. 스마트시티에서 건설관리의 위치
추가 2. 스마트시티 구현을 위한 건설관리 생애주기
추가 3. 스마트시티 건설관리 데이터 구조
추가 4. BIM에서 City Digital Twin으로의 확장
추가 5. 스마트시티 건설관리의 발전방향
추가 6. 본인 연구와 스마트시티 건설관리의 연결
```

6장을 모두 넣는 것을 추천한다. 시간이 부족하면 추가 6번은 기존 연구소개 또는 향후 연구계획 파트로 옮겨도 된다.

## 추가 1. 스마트시티에서 건설관리의 위치

### 슬라이드 제목

```text
스마트시티에서 건설관리의 위치
```

### 핵심 문장

```text
스마트시티는 ICT 서비스의 집합이 아니라,
도시를 구성하는 건축물과 인프라의 생애주기 데이터를
통합적으로 관리하는 체계이다.
```

### 본문

```text
스마트시티의 물리적 기반
• 건축물, 도로, 교량, 공공시설, 에너지 인프라
• 도시 서비스를 가능하게 하는 물리적 공간과 시설

건설관리의 역할
• 기획-설계-시공-운영 단계의 의사결정 통합
• 공정, 비용, 품질, 안전, 유지관리, 탄소, 사용자 영향 관리

핵심 기술
• BIM, AI, Digital Twin, IoT, GIS, 센서 데이터, 클라우드 협업
```

### 하단 강조문

```text
Project-level Construction Management
→ City-level Asset and Infrastructure Management
```

### 발표 멘트

```text
스마트시티는 교통, 에너지, 행정 서비스처럼 운영 단계의 ICT 서비스로 이해되는 경우가 많습니다. 그러나 실제 기반은 건축물, 도로, 공공시설, 에너지 인프라와 같은 물리적 공간입니다. 건설관리는 이 물리적 기반의 데이터를 기획, 설계, 시공, 운영 단계에서 생성하고 검증하며 연결합니다. 따라서 스마트시티에서 건설관리는 개별 프로젝트 관리에서 도시 단위 자산과 인프라 관리로 확장됩니다.
```

## 추가 2. 스마트시티 구현을 위한 건설관리 생애주기

### 슬라이드 제목

```text
스마트시티 구현을 위한 건설관리 생애주기
```

### 핵심 문장

```text
스마트시티의 성능은 기획-설계-시공-운영 단계의 데이터가
끊기지 않고 연결될 때 향상된다.
```

### 본문

| 단계 | 건설관리 역할 | 스마트시티 연계 |
|---|---|---|
| 기획 | 수요, 사업성, LCC, 리스크 검토 | 도시 수요, 공공성, 탄소, 사용자 영향 |
| 설계 | BIM/CIM 기반 정보모델 구축 | 에너지, 일조, 동선, 안전, 도시 데이터 연계 |
| 시공 | 4D/5D 기반 공정·원가·품질관리 | 드론, CCTV, IoT 기반 현장 모니터링 |
| 운영 | 6D/7D 기반 유지관리 | Digital Twin 기반 예측 유지관리·자산관리 |

### 하단 강조문

```text
건설관리는 스마트시티 데이터를 실제 도시공간으로 구현하고,
운영 가능한 자산으로 전환하는 연결 역할을 수행한다.
```

### 발표 멘트

```text
스마트시티에서 중요한 것은 운영 단계의 데이터만이 아닙니다. 기획 단계에서의 수요와 사업성, 설계 단계의 BIM 정보, 시공 단계의 공정과 품질 데이터, 운영 단계의 유지관리 이력이 하나의 생애주기로 연결되어야 합니다. 이 연결을 담당하는 분야가 건설관리입니다.
```

## 추가 3. 스마트시티 건설관리 데이터 구조

### 슬라이드 제목

```text
스마트시티 건설관리 데이터 구조
```

### 핵심 문장

```text
스마트시티 건설관리는 다양한 현장·시설·도시 데이터를
BIM과 Digital Twin을 중심으로 통합하는 데이터 기반 의사결정 체계이다.
```

### 본문

```text
Input Data
• BIM/CIM 모델
• 공정표, 원가, 계약, 품질 데이터
• 드론, CCTV, LiDAR, IoT 센서 데이터
• 유지관리 이력, 민원, 점검 기록
• GIS, 교통, 환경, 에너지 데이터

Integration Platform
• BIM + Digital Twin + AI Analytics

Decision Output
• 공정·원가 예측
• 안전 리스크 탐지
• 하자·노후도 예측
• 유지관리 우선순위
• 탄소·에너지 성능 관리
• 시민 안전 및 생활환경 영향 관리
```

### 도식 추천

```text
[Input Data] → [BIM / Digital Twin / AI Platform] → [Decision Making]
```

### 발표 멘트

```text
스마트시티 건설관리는 데이터를 많이 모으는 것 자체가 목적이 아니라, 의사결정 가능한 형태로 통합하는 것이 핵심입니다. BIM과 CIM 모델, 공정과 원가 데이터, 드론과 CCTV, IoT 센서 데이터, 유지관리 이력과 민원 데이터가 Digital Twin과 AI 분석 플랫폼으로 연결되면 공정지연, 안전위험, 하자, 노후도, 탄소와 에너지 성능까지 관리할 수 있습니다.
```

## 추가 4. BIM에서 City Digital Twin으로의 확장

### 슬라이드 제목

```text
BIM에서 City Digital Twin으로의 확장
```

### 핵심 문장

```text
BIM은 개별 건축물의 정보모델을 넘어
스마트시티 Digital Twin의 기초 데이터로 확장될 수 있다.
```

### 본문

```text
3D BIM
• 객체 기반 정보모델
• 설계정보, 물량, 위치, 속성정보

4D/5D BIM
• 공정 및 비용 정보 연계
• 시공단계 일정·자원·공사비 관리

6D/7D BIM
• 에너지, 유지관리, 자산관리 정보 연계
• 시설물 운영과 예방적 유지관리

City Digital Twin
• 건축물·인프라·도시운영 데이터 통합
• 도시 단위 안전, 에너지, 교통, 유지관리 의사결정 지원
```

### 하단 강조문

```text
스마트시티 관점에서 BIM은 설계도구가 아니라
도시 자산관리와 운영 의사결정의 데이터 기반이다.
```

### 발표 멘트

```text
BIM은 단순한 3차원 모델링 도구가 아닙니다. 3D BIM은 객체 정보를 담고, 4D와 5D BIM은 공정과 비용을 연결하며, 6D와 7D BIM은 에너지와 유지관리 정보까지 확장합니다. 이 정보가 도시의 GIS, 센서, 교통, 환경 데이터와 연결되면 City Digital Twin의 기초가 됩니다. 이 관점에서 뒤의 BIM 기반 가상건설 내용은 스마트시티 구현을 위한 기반기술로 이해할 수 있습니다.
```

## 추가 5. 스마트시티 건설관리의 발전방향

### 슬라이드 제목

```text
스마트시티 건설관리의 발전방향
```

### 핵심 문장

```text
스마트시티에서 건설관리는 사후 대응형 관리에서
예측형·실시간·사용자 중심 관리로 발전해야 한다.
```

### 본문

| 현재 건설관리 | 스마트시티 건설관리 |
|---|---|
| 프로젝트 단위 관리 | 도시 인프라 단위 관리 |
| 공정·원가 중심 | 안전·탄소·유지관리·사용자 영향 통합 |
| 사후 대응형 유지관리 | AI 기반 예측 유지관리 |
| 현장 중심 안전관리 | 작업자·시민·생활환경 리스크 통합관리 |
| BIM 모델 활용 | BIM + Digital Twin + AI 기반 의사결정 |

### 하단 강조문

```text
AI + BIM + Digital Twin 기반 통합 건설관리가
스마트시티의 지속가능한 운영을 가능하게 한다.
```

### 발표 멘트

```text
앞으로 스마트시티에서 건설관리는 프로젝트 단위 관리에서 도시 인프라 단위 관리로 확장되어야 합니다. 공정과 원가 중심의 관리를 넘어 안전, 탄소, 유지관리, 사용자 영향을 통합적으로 고려해야 합니다. 또한 사후 대응형 유지관리에서 AI 기반 예측 유지관리로 전환되어야 합니다. 결국 AI, BIM, Digital Twin 기반 통합 건설관리가 스마트시티의 지속가능한 운영을 가능하게 할 것입니다.
```

## 추가 6. 본인 연구와 스마트시티 건설관리의 연결

### 슬라이드 제목

```text
본인 연구와 스마트시티 건설관리의 연결
```

### 핵심 문장

```text
본인의 연구는 건설 프로젝트와 시설물 생애주기에서 발생하는
데이터 기반 의사결정 문제를 해결하는 방향으로 확장되어 왔다.
```

### 본문

| 본인 연구 | 스마트시티 건설관리 확장 |
|---|---|
| 건설소음 예측 및 리스크 평가 | 생활환경·민원·환경영향 관리 |
| 유지관리 비용 및 수명 예측 | 도시 자산관리 및 예측형 유지관리 |
| 웨어러블 기반 작업자 안전관리 | 스마트 현장 및 도시 안전관리 |
| BIM 기반 가상건설 | City Digital Twin 기반 시공·운영관리 |
| KoBERT·Transformer 기반 하자 및 유지관리 플랫폼 | 비정형 유지관리 데이터 분석 및 문서 자동화 |

### 하단 강조문

```text
Project Data → Facility Data → Urban Asset Data
```

### 발표 멘트

```text
저의 연구는 주제는 다양해 보이지만 공통적으로 건설 프로젝트와 시설물 생애주기에서 발생하는 데이터 기반 의사결정 문제를 해결하는 데 초점이 있습니다. 건설소음 연구는 스마트시티의 생활환경 관리와 연결되고, 유지관리 비용과 수명 예측은 도시 자산관리와 연결됩니다. 웨어러블 기반 안전관리와 BIM 기반 가상건설, KoBERT 기반 유지관리 플랫폼은 향후 스마트 현장관리와 City Digital Twin 기반 시설관리로 확장될 수 있습니다.
```

## 기존 내용으로 넘어가는 연결 멘트

추가 4번 또는 추가 6번 뒤에 기존 학력소개 슬라이드로 넘어갈 때 다음 멘트를 사용하면 자연스럽다.

```text
이러한 스마트시티 건설관리 관점에서, 제가 어떤 배경을 통해 BIM, 건설관리, AI 기반 스마트건설 연구를 수행해 왔는지 간단히 말씀드리겠습니다.
```

기존 BIM 기반 가상건설 슬라이드로 넘어갈 때는 다음 멘트를 사용한다.

```text
이제 스마트시티 건설관리의 기반기술 중 하나인 BIM 기반 가상건설과 건설관리 활용을 구체적으로 설명드리겠습니다.
```

## 시간 운영

추가 슬라이드 6장을 모두 사용할 경우:

```text
추가 1: 1분 20초
추가 2: 1분 20초
추가 3: 1분 30초
추가 4: 1분 20초
추가 5: 1분 20초
추가 6: 1분 30초
합계: 약 8분 20초
```

20분 발표에서는 추가 슬라이드 전체를 약 7분 30초에서 8분 정도로 운영하고, 기존 학력 및 BIM 슬라이드는 빠르게 넘어가는 것이 좋다.

## 최종 핵심 문장

```text
스마트시티에서 건설관리는 건축물과 인프라의 생애주기 데이터를
AI·BIM·Digital Twin 기반으로 연결하여,
도시의 안전성, 지속가능성, 운영 효율성을 높이는 핵심 의사결정 체계입니다.
```
