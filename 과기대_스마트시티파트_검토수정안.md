# 과기대 스마트시티 파트 검토 및 수정안

검토 대상: `과기대_스마트시티 파트.pdf`

## 1. 전체 판단

현재 흐름은 `개념 → 전환 필요성 → 기술 분류 → 5개 기술 상세 → Gap 분석 → 발전방향`으로 구성되어 있어 발표 구조는 유지해도 됩니다. 다만 발표자가 읽을 제목, 강조 문장, 하단 네모박스 문구는 일부 문장이 길고 명사형이 많아 말로 읽을 때 딱딱하게 들립니다.

가장 중요한 수정 방향은 다음과 같습니다.

- `연관`, `필요`, `구축 필요` 같은 느슨한 표현을 `확장된다`, `핵심 기반이다`, `전환되어야 한다`처럼 발표형 문장으로 바꾸기
- `Digital Twin`, `디지털트윈`, `디지털 Map`, `Digital Map`처럼 섞인 용어는 한 방식으로 통일하기
- 4~8쪽의 5개 기술은 유지하되, 각 장의 역할을 분명히 다르게 설명하기
- 15쪽은 문장이 중복되어 겹쳐 보이는 오류가 있으므로 반드시 수정하기

## 2. 5개 기술 구성 검토

현재 4~8쪽의 5개 기술은 완전히 중복되지는 않습니다. 그대로 두는 것이 좋습니다. 다만 지금은 `BIM`, `Digital Twin`, `데이터 통합`, `자동화`가 여러 장에서 반복되어 경계가 흐려 보일 수 있습니다.

권장 구분은 다음과 같습니다.

| 현재 페이지 | 권장 기술축 | 발표에서 강조할 차이 |
|---|---|---|
| 4쪽 | 현장 데이터 취득 및 디지털트윈 구축 | 현장 상태를 데이터로 만드는 출발점 |
| 5쪽 | 장비관제 및 시공 자동화 | 장비 운영, 작업경로, 품질을 실시간 최적화 |
| 6쪽 | 프리팹·인프라 BIM 기반 생산관리 | 설계-제작-운반-설치 정보를 연결해 오류와 재작업 감소 |
| 7쪽 | 로보틱스·원격시공 기반 고위험 공종 자동화 | 작업자를 위험 작업에서 분리하고 시공 정밀도 향상 |
| 8쪽 | AI 기반 예측형 안전관리 | 사고 후 대응에서 위험 예측과 즉시 대응으로 전환 |

따라서 3쪽의 기술 분류도 다음처럼 맞추는 편이 좋습니다.

```text
01 현장 데이터 취득·디지털트윈
02 장비관제·시공 자동화
03 프리팹·인프라 BIM
04 로보틱스·원격시공
05 AI 안전·예측 유지관리
```

이렇게 하면 4~8쪽이 각각 한 기술축에 대응하므로 중복으로 보이지 않습니다.

## 3. 페이지별 수정안

### 1쪽. 스마트시티와 건설관리의 역할

권장 제목:

```text
스마트시티에서 건설관리의 역할
```

수정할 표현:

- `계약 및 현업관리` → `계약 및 협업관리`
- `도시 운영을 위한 데이터 체계를 구축하는 것과 연관` → `도시 운영을 위한 데이터 체계 구축으로 확장됨`

하단 네모박스 권장 문구:

```text
스마트시티에서 건설관리는 프로젝트 단위를 넘어,
도시 인프라의 생애주기 의사결정을 지원하는 데이터 관리·운영 체계로 확장된다.
```

### 2쪽. 데이터 기반 건설관리 전환 필요성

권장 제목:

```text
데이터 기반 건설관리로의 전환 필요성
```

수정할 표현:

- `개선 필요` → `개선이 필요함`
- `선택적 기술 도입이 아니라`는 좋지만, 발표에서는 `선택사항이 아니라`가 더 자연스러움

하단 네모박스 권장 문구:

```text
스마트시티 구현에서 건설관리는 선택사항이 아니라,
생산성·안전성·운영 효율을 높이는 필수 기반이다.
```

### 3쪽. 스마트시티 관련 건설관리 기술

현재 1번 카드에 `현장 데이터 취득데이터`, `현장 기술 취득`처럼 중복되어 보이는 표현이 있습니다. 다음처럼 정리하는 것이 좋습니다.

```text
01 현장 데이터 취득·디지털트윈
02 장비관제·시공 자동화
03 프리팹·인프라 BIM
04 로보틱스·원격시공
05 AI 안전·예측 유지관리
```

하단 네모박스 권장 문구:

```text
스마트시티 건설관리는 개별 기술의 도입보다,
현장 데이터가 정보모델과 AI 분석을 거쳐 도시 운영 의사결정으로 환류되는 구조가 핵심이다.
```

### 4쪽. 현장 데이터 취득과 Digital Twin

권장 제목:

```text
현장 데이터 취득 및 디지털트윈 구축
```

수정할 표현:

- `스마트시티 구현을 위해서` → `스마트시티 구현을 위해`
- `전환 필요` → `전환해야 함`
- `고도화 할 필요` → `고도화할 필요`

하단 네모박스 권장 문구:

```text
현장 계측 데이터는 As-built BIM과 City Digital Twin으로 확장되어,
도시 인프라의 생애주기 의사결정을 고도화하는 출발점이 된다.
```

### 5쪽. 장비관제와 건설자동화

권장 제목:

```text
장비관제 및 시공 자동화
```

수정할 표현:

- `보완측위` → `보완 측위`
- `디지털 Map` → `디지털맵` 또는 `Digital Map` 중 하나로 통일
- `최적화 필요` → `최적화해야 함`

하단 네모박스 권장 문구:

```text
장비관제와 자동화의 핵심은 개별 장비의 자동화가 아니라,
장비·데이터·통신·관제를 실시간으로 연결해 작업계획과 품질 의사결정을 최적화하는 것이다.
```

### 6쪽. 디지털 기반 프리팹과 인프라 BIM

권장 제목:

```text
프리팹·인프라 BIM 기반 생산관리
```

수정할 표현:

- `2D 도면 기반한` → `2D 도면에 기반한`
- `프리패브`, `Prefab`, `프리팹` 중 하나로 통일. 발표자료에서는 `프리팹` 권장
- `간섭검토` → `간섭 검토`
- `클라우드 BIM/CDE 기반으로` → `클라우드 BIM/CDE를 기반으로`

하단 네모박스 권장 문구:

```text
프리팹·인프라 BIM은 설계-제작-운반-설치 데이터를 하나의 정보모델로 연결해,
시공 오류와 재작업을 줄이고 유지관리 데이터로 확장하는 기반이다.
```

### 7쪽. 로보틱스와 원격시공 자동화

현재 하단 문구의 `고위험 작업자 파악`은 어색합니다. `작업자를 파악`하는 것이 아니라 `작업자 투입을 줄이거나 위험 작업에서 분리`하는 것이 핵심입니다.

수정할 표현:

- `통합관리한계` → `통합관리 한계`
- `높이는데 있음` → `높이는 데 있다`
- `고위험 작업자 파악` → `고위험 작업에서 작업자 투입을 줄이고`

하단 네모박스 권장 문구:

```text
로보틱스·원격시공은 고위험 작업에서 작업자를 물리적으로 분리하고,
센서·AI 기반 제어로 시공 정밀도와 안전성을 높이는 기술 축이다.
```

### 8쪽. AI 기반 스마트 안전관리

수정할 표현:

- `작업자 안전 뿐 아니라` → `작업자 안전뿐 아니라`
- `사고 빌생 후` → `사고 발생 후`
- `Wearable Device와 3D camera` → `웨어러블 디바이스와 3D 카메라`
- `체계 구축`은 줄바꿈이 어색하므로 한 줄 안에서 정리

하단 네모박스 권장 문구:

```text
AI 기반 안전관리는 사고 발생 후 대응을 넘어,
영상·센서·디지털트윈 데이터를 결합해 위험을 예측하고 즉시 대응하는 예방형 안전관제로 전환되어야 한다.
```

만약 3쪽에서 `안전 및 유지관리`라고 유지할 경우, 8쪽 제목은 다음처럼 바꾸는 것이 더 일관됩니다.

```text
AI 기반 스마트 안전·유지관리
```

그리고 기술 항목에 `시설물 이상징후 및 임시구조물 상태 예측`을 추가하는 것이 좋습니다.

### 9쪽. 국내·외 스마트시티-건설관리 기술 Gap 분석

이 슬라이드는 정보량이 많으므로 발표자는 하단 네모박스 위주로 읽고, 표는 근거 자료로 보여주는 방식이 적합합니다.

권장 제목:

```text
국내·외 스마트시티 건설관리 기술 격차 분석
```

수정할 표현:

- `미보유 (추진 無)` → `미보유 또는 미추진`
- `거버먼트` → `공공서비스/거버넌스`
- `Gap 분석`은 가능하지만, 국문 발표에서는 `기술 격차 분석`이 더 자연스러움

하단 네모박스 권장 문구:

```text
국내 스마트시티 기술은 영역별 보유 수준의 편차가 크고,
건설 단계 데이터가 도시 운영 데이터로 이어지지 못하는 데이터 단절이 핵심 한계이다.
```

### 10쪽. 스마트건설관리 기술의 한계와 개선방향

수정할 표현:

- `예측형 공정 · 품질 안전관리` → `예측형 공정·품질·안전관리`
- `장비 센서 플랫폼 표준 부족` → `장비·센서·플랫폼 표준 부족`
- `시민 안전 및 탄소 회복 탄력성` → `시민 안전·탄소저감·회복탄력성`
- `연구 교육 산학협력 플랫폼` → `연구·교육·산학협력 플랫폼`

하단 네모박스 권장 문구:

```text
현장 데이터는 정보모델과 AI 의사결정을 거쳐 도시 운영으로 연결되고,
그 결과가 다시 현장 관리로 환류되는 폐쇄루프형 관리체계가 되어야 한다.
```

### 11쪽. 스마트시티 건설관리 발전방향

수정할 표현:

- `디지털 전환이 열세하고` → `디지털 전환 수준이 낮고`
- `핵심 기술의 미보유 · 데이터 분절` → `핵심 기술 미보유와 데이터 분절`
- `다수 미 보유` → `다수 미보유`
- `영역 간 데이터 연계 ·통합` → `영역 간 데이터 연계·통합`

하단 네모박스 권장 문구:

```text
스마트 건설관리는 데이터 취득·정보모델·통합 플랫폼을 연결해,
영역별 기술 미보유와 데이터 분절의 한계를 해소하는 핵심 역할을 수행한다.
```

### 15쪽. 강의계획 및 주안점

이 페이지는 문장이 중복되어 겹친 흔적이 있어 반드시 수정해야 합니다.

현재 문제:

- `• •BIM을 ...`처럼 불릿이 중복됨
- `건축물을 시공하기 시공하기`처럼 문구가 반복됨
- `현장적용공법과시공를`은 문법 오류
- 같은 의미의 문장이 위아래로 반복됨

권장 본문:

```text
• BIM을 활용하여 건축물을 설계·시공하기 위한 실질적인 방법을 습득한다.
• 물량산출, 간섭 체크, 에너지 부하 시뮬레이션 등 공학적 적용 방법의 기본 소양을 연마한다.
• 현장 적용 공법과 시공계획을 연계한 팀 프로젝트를 수행한다. (1조 2~3인)
```

하단 네모박스 권장 문구:

```text
BIM의 핵심 개념인 협업 프로세스를 이해하고,
공학 분야에서 BIM을 활용하는 실질적인 방법을 학습한다.
```

### 16쪽. 강의계획 및 주안점

수정할 표현:

- `학부과정동안` → `학부 과정 동안`
- `1조: 4-5명` → `1조 4~5명`
- `종합적, 실무적 능력` → `종합적·실무적 역량`

권장 하단 문구:

```text
BIM과 Construction IT를 활용한 Virtual Construction 역량을
종합적·실무적으로 향상시키는 것을 목표로 한다.
```

### 17쪽. 향후 연구계획

수정할 표현:

- `예산: 25억/년` → `예산: 연 25억 원`
- `제안서 및 발표자료의 기획 및 작성 총괄` → `제안서와 발표자료 기획·작성 총괄`
- `참여, 기여할 것` → 발표에서는 `참여하여 기여하겠습니다`가 자연스러움

권장 문구:

```text
“한양대학교 조기취업형 계약학과 선도대학원 육성사업” 참여 및 최종 선정
(2024.5, 예산: 연 25억 원)

PM: 제안서와 발표자료 기획·작성 총괄

서울과학기술대학교에서도 대학·학과 차원의 산학협력 및 교육사업 기획에 참여하여,
학과 발전과 실무형 인재 양성에 기여하겠습니다.
```

## 4. 발표자료에 더 적합한 용어

| 현재 표현 | 권장 표현 |
|---|---|
| 연관 | 직결됨, 확장됨, 연결됨 |
| 필요 | 필요함, 전환되어야 함, 구축해야 함 |
| 선택적 기술 도입 | 선택사항, 부가 기능 |
| 필수 요소 | 필수 기반, 핵심 기반 |
| 건설자동화 | 시공 자동화 |
| Digital Twin / 디지털트윈 혼용 | `디지털트윈`으로 통일하거나 첫 등장만 `Digital Twin(디지털트윈)` |
| 디지털 Map | 디지털맵 또는 Digital Map 중 하나로 통일 |
| 프리패브 / Prefab / 프리팹 | `프리팹`으로 통일 |
| Gap | 격차, 기술 격차 |
| 거버먼트 | 공공서비스, 거버넌스 |
| 폐쇄루프 | 환류형 또는 폐쇄루프형(Closed-loop) |

## 5. 최종 발표용 핵심 메시지

발표 전체를 관통하는 문장은 다음으로 잡으면 좋습니다.

```text
스마트시티에서 건설관리는 단순한 시공관리나 개별 기술 도입이 아니라,
현장 데이터와 BIM·디지털트윈·AI를 연결해 도시 인프라의 생애주기 의사결정을 지원하는 핵심 관리체계이다.
```

4~8쪽을 소개할 때는 다음처럼 말하면 중복이 줄어듭니다.

```text
첫째, 현장 데이터 취득과 디지털트윈은 현장 상태를 데이터화하는 출발점입니다.
둘째, 장비관제와 시공 자동화는 장비 운영과 품질을 실시간으로 최적화합니다.
셋째, 프리팹·인프라 BIM은 설계-제작-시공 데이터를 연결해 생산성과 품질을 높입니다.
넷째, 로보틱스와 원격시공은 고위험 작업에서 작업자 안전과 시공 정밀도를 확보합니다.
다섯째, AI 기반 안전관리는 사고 후 대응을 넘어 위험을 예측하고 즉시 대응하는 체계로 전환합니다.
```
